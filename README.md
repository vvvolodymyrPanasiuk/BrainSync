# BrainSync

**Local AI-powered personal knowledge management via Telegram.**

Send a message, voice note, photo, PDF, or YouTube link вҶ’ AI classifies, formats, and saves it as a structured Markdown note in your Obsidian vault. Ask questions about your notes in natural language вҖ” the bot searches your vault semantically and answers using only your own knowledge. Everything runs locally, nothing leaves your machine except optional AI API calls.

## What it does

BrainSync solves two problems: **capture** and **retrieval**.

**Capture** вҖ” you have a thought while doing something else. Open Telegram, send a message, and it appears in your vault as a properly formatted note вҖ” correct folder, frontmatter, wikilinks to related notes, added to the Map of Content. Zero manual filing.

**Retrieval** вҖ” you remember writing something about a topic but can't find it. Ask the bot in plain language: *"СүРҫ СҸ РҙСғРјР°РІ РҝСҖРҫ CQRS?"* вҖ” it finds semantically relevant notes and gives you a synthesized answer with citations, even if your notes never contained the exact words you typed.

## Features

### Capture

| What you send | What happens |
|---------------|--------------|
| Plain text message | AI semantic router вҶ’ classifies intent вҶ’ formats вҶ’ saves |
| Text with inline prefix (`task:`, `С–РҙРөСҸ:`) | Forced type, skips AI routing |
| Voice message | On-device transcription (Whisper) вҶ’ note |
| Photo | AI visual description вҶ’ note |
| PDF file | Full text extracted locally вҶ’ note |
| `.txt` / `.md` file | Content saved directly as note |
| YouTube URL | NotebookLM session вҶ’ Q&A вҶ’ save to vault |
| Any bare URL | Web page fetched, AI-summarised вҶ’ note |

### Intelligence

- **AI Semantic Router** вҖ” every plain-text message goes through a single AI call that returns an `ActionPlan`: intent, target folder (4-level hierarchy), note type, tags, title, and whether to save, search, or answer.
- **Semantic search** вҖ” `/search` and natural language queries use vector embeddings; finds notes by meaning, not exact words. Works in Ukrainian.
- **RAG answers** вҖ” questions about your vault get synthesized answers grounded exclusively in your own notes, with source citations.
- **Duplicate detection** вҖ” after every note save, checks for semantically similar existing notes (вүҘ 85% similarity) and offers to merge.
- **Smart note splitting** вҖ” if a message covers multiple unrelated topics, AI automatically splits it into separate notes.
- **Web clipping** вҖ” paste any URL; the bot fetches the page and AI-summarises it into a structured vault note.
- **YouTube Г— NotebookLM** вҖ” paste a YouTube URL to open an interactive Q&A session powered by NotebookLM; save the session as a vault note when done.
- **Knowledge gap analysis** вҖ” `/gaps <topic>` asks AI to identify missing subtopics in your vault for a given subject.

### Vault management

- **Map of Content** вҖ” index files auto-updated every time a note is added to a topic
- **4-level folder hierarchy** вҖ” `GeneralCategory/Topic/Subtopic/Section`
- **Auto wikilinks** вҖ” related notes automatically linked to each other on save
- **Vault health check** вҖ” `/health` reports orphan notes, broken links, missing aliases, and potential duplicates
- **Note merge** вҖ” merge a new note with an existing duplicate via inline button (with confirmation dialog)
- **Note move** вҖ” move notes to different folders via `/move` command or inline button after save
- **Tag management** вҖ” add tags to any saved note via inline button
- **Group Topics** вҖ” map Telegram Forum Topic threads to vault folders; `/register-topic` once per thread

### Gamification

- **XP system** вҖ” earn XP for every note saved
- **Streaks** вҖ” daily streak tracking with notifications at 3, 7, 14, 30, 100 days
- **Milestones** вҖ” achievements at 10, 25, 50, 100, 250, 500, 1000 notes
- **Levels** вҖ” Beginner вҶ’ Note Taker вҶ’ Chronicler вҶ’ Knowledge Builder вҶ’ Archivist вҶ’ Vault Master вҶ’ Grand Sage

### Analytics & Visualization

- **`/stats`** вҖ” total notes, per-folder bar chart, 30-day activity line chart, XP/streak; sends as PNG chart if matplotlib installed
- **`/graph`** вҖ” knowledge graph PNG showing wikilink connections between notes, colored by folder; requires `networkx` + `matplotlib`
- **Scheduled summaries** вҖ” daily text digest; weekly and monthly reports include 2вҖ“3 panel PNG charts (notes by topic, daily activity, type breakdown, month-over-month comparison)

### Automation

- **Git sync** вҖ” vault auto-committed and pushed after every note (configurable interval)
- **Stale task reminder** вҖ” configurable daily reminder for tasks open longer than N days
- **Hot reload** вҖ” `/reload` reloads `config.yaml` without restarting the bot
- **MCP server** вҖ” exposes vault operations to Claude Code sessions

### UX

- **Progress indicator** вҖ” `вҸі ThinkingвҖҰ` message shown during AI processing, deleted when result arrives
- **Inline keyboards** вҖ” actionable buttons after every note save: [рҹ“Ғ Move] [рҹҸ·пёҸ Tags]; duplicate detected: [рҹ”Җ Merge] [вң… Keep both]
- **Merge confirmation** вҖ” destructive merge operation always requires [вң… Confirm] [вқҢ Cancel]
- **`/settings`** вҖ” inline keyboard to toggle auto-commit, wikilinks, MoC, daily summary; changes persist to `config.yaml`
- **`/today`** вҖ” today's notes + all open tasks at a glance

### Infrastructure

- **Fully offline capable** вҖ” sentence-transformers for embeddings, Ollama for AI, Whisper for voice
- **Vector index persists** вҖ” ChromaDB stores embeddings in `data/chroma/`; survives restarts
- **Background indexing** вҖ” vault indexed at startup without blocking the bot

Requires `ollama_vision_model` (e.g. `llava`) or an Anthropic model.

### PDFs
Full text extracted locally via `pypdf` вҶ’ first 3 000 chars sent to AI for classification вҶ’ full text saved to vault.
Limits: `media.pdf_max_pages` (default 50), `media.max_file_size_mb` (default 20).

### Plain text / Markdown files
Attach `.txt` or `.md` вҶ’ content saved as note directly.

### YouTube URLs
Send a bare YouTube URL вҶ’ BrainSync creates a NotebookLM notebook, adds the video as source, and enters interactive Q&A mode. Each message in the session is answered by NotebookLM. Press **рҹ’ҫ Save to vault** to save the session as a note; the notebook is deleted after save. Requires `notebooklm-py`.

### Bare URLs (web clip)
Send any `https://...` URL that isn't YouTube вҶ’ BrainSync fetches the page, extracts text, AI-summarises into a structured note, and saves it.

## AI Semantic Router

Every plain-text message (without prefix) goes through a single AI call that returns an `ActionPlan`:

| Field | Example |
|-------|---------|
| `intent` | `CREATE_NOTE`, `ANSWER_FROM_VAULT`, `SEARCH_VAULT`, `MOVE_NOTE`, вҖҰ |
| `target_folder` | `"Technology/Python"` |
| `note_type` | `"note"`, `"task"`, `"idea"`, `"journal"` |
| `tags` | `["python", "async"]` |
| `title` | `"Asyncio event loop internals"` |
| `should_save` | `true` / `false` |

Intents and what happens:

| Intent | Behaviour |
|--------|-----------|
| `CREATE_NOTE` | Format вҶ’ enrich with wikilinks вҶ’ write вҶ’ MoC вҶ’ duplicate check |
| `ANSWER_FROM_VAULT` | RAG: vector search вҶ’ AI synthesizes answer from your notes |
| `SEARCH_VAULT` | Semantic search вҶ’ ranked results list |
| `CHAT_ONLY` | General conversation; with `claude_code` provider вҶ’ uses web search automatically |
| `SEARCH_WEB` | Web search via Claude Code tools вҶ’ AI-synthesized answer with sources |
| `MOVE_NOTE` | Find note semantically вҶ’ move to target folder |
| `APPEND_NOTE` | Find closest note вҶ’ append content |
| `UPDATE_NOTE` | Find closest note вҶ’ rewrite with new info |
| `REQUEST_CLARIFICATION` | Ask one follow-up question; next reply resolves the action |
| `IGNORE_SPAM` | Silently discard |

## Duplicate detection & merge

After every save, BrainSync checks cosine similarity against existing notes:

- **вүҘ 85%** вҖ” duplicate warning + [рҹ”Җ Merge] [вң… Keep both] buttons
- **70вҖ“84%** вҖ” related note suggestion

Clicking **Merge** shows a confirmation dialog. On confirm, AI merges the two notes (preserves frontmatter from the existing note, deduplicates body), deletes the new file, and updates the vector store.

Thresholds configurable in `config.yaml` under `embedding`.

## Note format

Every note created by BrainSync:

```markdown
---
title: "CQRS Pattern"
date: 2026-03-30
categories: [Architecture]
tags:
  - areas/architecture
  - types/note
MoC: "[[0 Architecture]]"
aliases:
  - CQRS Pattern
---

## Description

CQRS (Command Query Responsibility Segregation) separates the read and write
models of an application. Commands mutate state, queries return data.

## Conclusions

Use CQRS when read and write operations have significantly different scaling
requirements. Pairs naturally with Event Sourcing.

## Links

- [[0003 Event Sourcing]]
- [[0001 DDD Basics]]
```

**Naming:** `NNNN Title.md` вҖ” 4-digit zero-padded sequential number per folder.
**Location:** `<vault>/<GeneralCategory>/<Topic>/<Subtopic>/<Section>/_data/NNNN Title.md`
**MoC files:** `0 TopicName.md` вҖ” auto-created at each folder level.

## Security

- `config.yaml` is in `.gitignore` вҖ” never committed
- `data/chroma/` is in `.gitignore` вҖ” vault embeddings never leave the machine
- `api_key` and `bot_token` are never written to logs or AI prompts
- `log_ai_decisions: true` logs only classification metadata вҖ” never note content or query text
- All Telegram interactions silently rejected for user IDs not in `allowed_user_ids`
- Path traversal guard in vault writer вҖ” notes can only be written inside `vault.path`
- `/settings` toggle persists only boolean flags to `config.yaml` вҖ” no arbitrary writes
